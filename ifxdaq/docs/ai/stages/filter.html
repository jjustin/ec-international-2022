Filter stage

The filters improve the quality of the tracked labels, which were generated with the Tracking stage.

Available stages

Stage name

Description & Parameters (API)

crop_lag
ifxdaq.ai.filter.stage.crop_lag(label, output_name=None)

Crop the lag at the end of tracks in the given LABEL_DIR(s).

The lag is caused by the hold gate introduced during Kalman filtering, which is needed to connect partially occluded or short-time hidden objects.

Parameters

  • label (Sequence[Path]) – Label dir(s) to process.

  • output_name (Optional[str]) – Optional individual name for the output directory.

Return type

List[Path]

Returns

List of folders with processed data.

merge_tracks
ifxdaq.ai.filter.stage.merge_tracks(label, threshold_metric, threshold_time, output_name=None)

Merge tracks in the given LABEL_DIR(s).

Consecutive tracks with different ID(s) that belong to the same person based on bounding box similarity are merged. Gaps between merged tracks are filled with a linear interpolation.

Parameters

  • label (Sequence[Path]) – Label dir(s) to process.

  • threshold_metric (float) – Threshold for the applied metric between two tracks.

  • threshold_time (float) – Maximum time between two tracks (in seconds).

  • output_name (Optional[str]) – Optional individual name for the output directory.

Return type

List[Path]

Returns

List of folders with processed data.
